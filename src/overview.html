MARCer

MARCer is an application written in Java that processes MARC 21 files.

MARCer reads a list of simple commands that allows the user to modify and write MARC files.

Commands and Syntax

The organization of the language follows the form of 'subject verb predicate'. For example printing out all the 008 tags from a MARC file is done with
008 print
But, of course, this MARCer is much more powerful.

  • [tag] [append|pre-pend] [some string] Eg:
  • [tag|'all'] print
  • [tag] if [position] == [value] then [instruction]
  • [tag] set [position] = [value]
  • set var [name] = [value] See Built-in variables.
  • record[s] write [file] as [binary|text] Outputs either each record as a single file or all records as one MARC file.

Built-in variables

Current built-in variables are 'ignore_white_space' and 'debug'.

ignore_white_space test the position of a character within a field, but will ignore whitespace. This means you are checking the position at starting at 0 on the first non-white space character. Example: ' hello', with ignore_white_space = false, 035 0 == h then 035 print will produce nothing, but ignore_white_space = true, 035 0 == h then 035 print will produce 'hello'. This setting is used for all comparisons and when ever a position is changed. It is generally not recommended.

debug turns debugging information on during run time. There is a secondary debug setting available when invoking MARCer from the command line.

Command line arguments

MARCer can be invoked from the command line with the following options.

  • -d Turn on debugging for general operations.
  • -f[marc_file.mrc] MARC file to be processed.
  • -i[instruction_file.cmd] File of modification instructions.
  • -v Print version and exit.
